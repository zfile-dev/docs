# 忘记管理员密码怎么办？

5.0.0 及以上版本提供本地 `reset-admin` 救援命令。它只接受本机回环请求，不需要开启 debug 模式；执行时 ZFile 主服务必须正在运行。

省略密码时会生成并打印一个随机强密码，同时关闭管理员的图形验证码、Turnstile 和双因素认证，并使旧登录会话失效。

## Linux 压缩包部署

如果通过一键脚本安装，在脚本所在目录执行：

```bash
./install.sh reset-admin
```

也可以指定新的用户名和密码：

```bash
./install.sh reset-admin --username admin --password '你的新密码'
```

手工解压捐赠版安装包时，在安装目录执行：

```bash title="捐赠版"
./bin/reset-admin.sh
```

开源版或没有包装脚本时，可直接调用二进制：

```bash title="开源版"
./zfile/zfile reset-admin
```

## Docker 部署

```bash title="开源版"
docker exec -it zfile /root/zfile reset-admin
```

```bash title="捐赠版"
docker exec -it zfile-pro /root/zfile-pro reset-admin
```

如果修改过容器名，请把命令中的 `zfile` 或 `zfile-pro` 换成实际容器名。

## Windows 部署

确保 ZFile 正在运行，然后双击解压目录中的 `重置管理员密码.bat`。也可以在命令提示符中执行该脚本并追加 `--username`、`--password` 参数。

## 提示未找到入口文件

先确认主服务正在运行，并在安装目录中执行命令。修改过数据目录或程序目录不可写时，可通过 `--data-path` 指定入口文件所在目录：

```bash
./bin/reset-admin.sh --data-path /你的/ZFile数据目录
```

旧版本仍可使用 [Debug 模式](/config/config-debug)重置，但该方式会临时暴露公网重置入口，建议升级后使用本地救援命令。
