# 启动失败 / 无法访问怎么办？

先在服务器本机执行：

```bash
curl -I http://127.0.0.1:8080
```

- 返回 HTTP 响应，说明 ZFile 已启动；外部仍无法访问通常是防火墙、安全组、端口映射或反向代理问题。
- 返回 `Connection refused`，说明端口没有监听，需要检查服务状态和日志。

如果修改过端口，请将 `8080` 换成实际端口。

## 使用一键脚本诊断

通过官方一键脚本安装时，在脚本所在目录执行：

```bash
./install.sh status
./install.sh doctor
./install.sh logs -n 200
```

Docker 手工部署可执行：

```bash
docker ps -a --filter name=zfile-pro
docker logs --tail 200 zfile-pro
```

5.0 统一安装包的默认容器名为 `zfile-pro`；从旧开源版升级且未重建容器时，容器名可能仍为 `zfile`。

Linux 压缩包部署可在安装目录执行 `./bin/status.sh` 和 `./bin/log.sh`。默认日志目录为 `~/.zfile-v4/logs`。

## 不需要单独安装 JDK

当前官方 Linux / Windows 安装包和 Docker 镜像已经包含运行所需环境，不再要求安装 JDK 8。不要使用旧教程中的 `java -jar` 或 JDK 8 启动方式，直接使用安装包自带的启动脚本。

## 端口冲突

默认端口为 `8080`。Linux 启动脚本会在启动前检查端口占用，并提示修改 `application.properties` 中的 `server.port`。也可以手动检查：

```bash
ss -ltnp | grep ':8080'
```

Docker 部署需要确保宿主机端口没有被占用，并检查 `-p 宿主机端口:8080` 是否正确。

## 数据目录权限

确认运行 ZFile 的用户对数据库和日志目录有读写权限。Docker 部署重点检查映射到 `/root/.zfile-v4/db`、`/root/.zfile-v4/logs` 的宿主机目录。

## 防火墙和安全组

云服务器通常需要同时在操作系统防火墙和云厂商安全组放行端口。Docker 映射或本机 `curl` 正常但外网无法访问时，优先检查这两处。

```bash title="firewalld"
firewall-cmd --zone=public --add-port=8080/tcp --permanent
firewall-cmd --reload
```

```bash title="ufw"
ufw allow 8080/tcp
```

如果通过 Nginx、1Panel 或宝塔反向代理，请同时确认代理目标是 ZFile 的实际监听地址和端口。
