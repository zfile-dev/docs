# 系统监控

「系统监控」是免费功能，无需填写授权码。进入后台「仪表盘」，即可查看系统监控、运行环境、挂载映射和构建信息。

## 系统资源监控

实时展示当前服务器和 ZFile 进程的核心资源指标：

- CPU 使用率与系统负载
- 系统内存与 ZFile 进程内存
- 网络实时上行 / 下行速率
- ZFile 进程 CPU、运行时长和线程等信息

默认展示近 10 分钟的高频趋势，也可以查看 1 小时、6 小时、24 小时、全部或自定义时间范围。历史趋势保存在内存中，默认最多保留 7 天，重启 ZFile 后会清空。

监控图表支持趋势平滑、暂停刷新和导出。

## 容器环境与挂载映射

ZFile 会自动识别 Docker、Podman、Kubernetes、containerd、LXC 等容器环境，并检查数据库、日志、本地存储等关键路径是否已正确持久化。

挂载表会区分：

- bind 挂载：宿主机目录直接映射进容器
- 匿名卷：没有固定名称的数据卷，重建容器时容易误丢
- 命名卷：由 Docker 管理的具名数据卷

基础容器识别和关键路径检查不依赖 `docker.sock`。即使不挂载，CPU、内存、网络趋势和容器内挂载检测仍可正常使用。

## 容器元数据

查看当前容器的名称、镜像、运行状态、健康状态、重启策略和宿主机挂载源路径，需要把宿主机的 `docker.sock` 只读挂载到容器中。

[安装向导](/install/guide)和一键安装脚本生成的 Docker 配置会默认添加此挂载。手工部署时请添加：

```bash title="docker run"
-v /var/run/docker.sock:/var/run/docker.sock:ro
```

```yaml title="docker compose"
volumes:
  - '/var/run/docker.sock:/var/run/docker.sock:ro'
```

:::caution
`docker.sock` 是高权限接口。`:ro` 只能防止容器替换 socket 文件，并不会把 Docker API 限制为只读调用，请仅使用可信镜像。
:::

修改挂载后需要重建容器，仅执行容器内重启不会增加新的挂载。

## 构建与运行信息

仪表盘还会展示后端版本、Git 提交、构建时间、数据库类型、数据库和日志目录大小等信息，便于确认运行版本及排查部署问题。
